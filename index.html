Node Strict Mode

What is it?

  • Javascript (and node) strict mode places a certain piece of code into "strict" javascript.
  • This means that "bad" syntax / silent errors which was previously accepted in javascript, will now throw an error.

// This will not cause an error.
x = 3.14;

// This will cause an error
"use strict";
x = 3.14;

Why would you ever want to use this?

Advantages:

  • It prevents accidental global leakage
  • global leakage = global variables assigned to asynchronous functions
  • It prevents, or throws errors, when relatively "unsafe" actions are taken (such as gaining access to the global object).
  • It turns silent mistakes into errors
  • It catches some common coding bloopers, throwing exceptions.
  • keywords such as: implements, interface,let, package, private, protected, public, static & yield are reserved. So you cant use it as a variable name
  • It disables features that are confusing or poorly thought out.

Disadvantages:

  • It removes octal literals (Integers can be expressed in decimal (base 10), hexadecimal (base 16), octal (base 8) and binary (base 2)
